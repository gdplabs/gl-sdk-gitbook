# Model Context Protocol

{% hint style="info" %}
This is an advanced topic covering how to leverage GL Connectors in all their forms: **as a REST API, Agentic Tools, and Model Context Protocol servers**. The following are quick references for understanding how GL Connectors authenticate and integrate with third-party services:

* [credentials.md](../api/credentials.md "mention")
* [http-plugin-concepts.md](../api/plugin-concepts/creating-custom-plugins/http-plugin-concepts.md "mention"), especially [#concept-5-authentication](../api/plugin-concepts/creating-custom-plugins/http-plugin-concepts.md#concept-5-authentication "mention")
* [#setting-up-an-integration](../api/quickstart.md#setting-up-an-integration "mention") or [custom-configurations](../api/plugin-concepts/custom-configurations/ "mention")
{% endhint %}

## GL Connectors' Output Types

GL Connectors as a service provides three types of accessible endpoints:

* **RESTful API** — reference documentation available [here](https://connectors.glair.ai/docs)
* **Agentic Tools** — compatible with GL SDK's [BaseTool](../../../gen-ai-sdk/tutorials/inference/lm-invoker/#tool-definition) and LangChain Tools (accessible via [GL Connector SDK](../api/agentic-tools-generator.md))
* **Model Context Protocol Servers** — usable by any MCP client, though we recommend our [in-house MCP Client](../../../common-modules/tutorials/tools/mcp-client.md).

The documentation below lists all tools provided by our MCP Servers and Tool Generator, along with the Connector responsible for each.

## Pre-Requisites

{% hint style="warning" %}
GL Connector Tools and MCP **does not handle setup and integration initialization**. It must be done independently outside the MCP itself either using the API, SDK, or CLI. GL Connectors MCP only provides an interface of interacting with the Connector API!
{% endhint %}

1. You will need to have [**Client Key**](../api/credentials.md#client-api-key) and [**User Token**](../api/credentials.md#user-token) (click the links to understand how to retrieve them). Or for quickstart, you may obtain them via the [connectors-console.md](../api/tools-and-interfaces/connectors-console.md "mention").
   1. **You&#x20;**_**must**_**&#x20;log into the account following the** [credentials.md](../api/credentials.md "mention") guide first. Neither the MCP nor Agentic Tools allow authentication nor integration management.
2. That account **must have** an appropriate integration.
   1. Like before, **you must create the integration outside tool call or MCP connection**. There isn't a way to provide integration initialization functionalities.
   2. Either use the [#integration-management](../api/tools-and-interfaces/cli.md#integration-management "mention") on the CLI, or [#setting-up-an-integration](../api/quickstart.md#setting-up-an-integration "mention") on the SDK, or utilize the [connectors-console.md](../api/tools-and-interfaces/connectors-console.md "mention").
3. **For MCP**: A client that supports either connecting to a Streamable HTTP MCP Server, _or_ able to run `mcp-remote` or `mcp-proxy` that allows unsupported clients to connect to Streamable HTTP MCP Servers.
